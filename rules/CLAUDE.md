# rules/

The four Semgrep rule files that make up the no-animal-violence detection suite. All files are auto-generated by `project-compassionate-code` — do not edit them directly.

## Files

| File | Languages | Match Target | Autofix | Rule count |
|------|-----------|-------------|---------|------------|
| `animal-violence-generic.yaml` | All (generic mode) | Comments, strings, docs, configs — any text | No | 62+ |
| `animal-violence-python.yaml` | Python | String literals (AST-aware via `metavariable-regex`) | Yes (`fix-regex`) | Same patterns as generic |
| `animal-violence-javascript.yaml` | JavaScript, TypeScript | String literals (AST-aware) | Yes (`fix-regex`) | Same patterns as generic |
| `animal-violence-go.yaml` | Go | String literals (AST-aware) | Yes (`fix-regex`) | Same patterns as generic |

## Why Both Generic and Language-Specific?

Generic rules use `languages: [generic]` (plain-text regex). They catch speciesist language everywhere — comments, docs, config files — but cannot apply autofixes.

Language-specific rules use Semgrep's AST parser to target only string literals. This is more precise and supports `fix-regex` autofix, but Semgrep's AST mode excludes comments. Run both for full coverage.

## Rule Structure

Every rule follows this shape:

```yaml
- id: animal-violence.<idiom-slug>           # generic
  # or: animal-violence.<lang>.string.<idiom-slug>   # language-specific
  pattern-regex: <regex>                     # generic only
  # or patterns + metavariable-regex:        # language-specific
  message: 'Animal violence language: "<phrase>". Consider: "<alt1>", "<alt2>".'
  languages: [generic]                       # or [python] / [javascript, typescript] / [go]
  severity: ERROR | WARNING | INFO
  metadata:
    category: inclusive-language
    subcategory: animal-violence
    alternative: <primary alternative>
    references:
      - https://doi.org/10.1007/s43681-023-00380-w
  fix-regex:                                 # language-specific rules only
    regex: <pattern>
    replacement: <replacement>
```

## Severity Guide

- **ERROR** — Direct violent idioms (e.g., the phrase that means accomplish two things at once, or belabor the point). Block CI by default.
- **WARNING** — Speciesist idioms and harmful tech jargon (e.g., "cattle vs. pets", "master/slave"). Flag for review.
- **INFO** — Deeply embedded jargon where alternatives exist but adoption is still growing (e.g., "pet project", "red herring"). Awareness only.

## Updating Rules

Rules are generated from the canonical dictionary in `project-compassionate-code`. To add or change a pattern:

1. Update `project-compassionate-code` canonical dictionary
2. Re-run the generator
3. Validate: `semgrep --config rules/ --validate`
4. Run structural tests: `pytest tests/`
